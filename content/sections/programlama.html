Proje Yapısı

Bu bölümde kodun nerede ve nasıl duracağını anlatıyoruz. Amaç; takımca okunur, bakım yapılabilir ve tekrarlanabilir bir iskelet oluşturmak.

Dosya/dizin yerleşimi

  • Uygulama (sketch): Arduino .ino dosyanız; proje giriş noktası.
  • Kütüphane: src/probot/... altında kontrolcüler, cihaz soyutlamaları ve IO bulunur.
  • Örnekler: File → Examples → Probot → hızlı başlangıç ve referans.

Başlangıç iskeleti

// Parola: DS erişimi için zorunlu
PROBOT_SET_DRIVER_STATION_PASSWORD("Guv3nliParola123");

// Global nesneler: motor, enkoder, PID, şasi vb.
// ClosedLoopMotor left(...), right(...);
// BasicTankDrive chassis(&left, &right);

void robotInit(){
  // Kalibrasyon/konfigürasyon (teker çevresi, track width, PID slotları)
}

Modülerlik ve soyutlamalar

  • Controllers: ClosedLoopMotor, BasicTankDrive gibi tekrar kullanılabilir bileşenler.
  • Devices: IMotor, MotorHandle, grup yapıları ve sahiplik (claim/release).
  • IO: joystick_api; mapping seçimi teleopInit()’te yapılır.

Bağımlılıklar ve sürücüler

  • Yer tutucu sürücüleri (Null) gerçek sürücülerle değiştirin (motor/enkoder veri sayfasına göre).
  • Harici kütüphaneleri Library Manager ile yönetin (örn. Adafruit NeoPixel).

Konfigürasyon ve sabitler

  • Mekanik sabitler: teker çevresi, iz genişliği; PID slot profilleri, timeout’lar.
  • Öneri: küçük bir config.hpp veya dosya üstü const sabitler.

Kayıt ve hata ayıklama

  • Serial Monitor ile minimal, anlamlı log; kritik değerleri geçici yazdırın.
  • Periyot takibi (≈20 ms) ve yoğun anlarda log seviyesini azaltma.

Robot Yaşam Döngüsü

Bu bölümde ne zaman hangi kodun çalıştığını anlatıyoruz. Driver Station ile mod değişimleri, kullanıcı geri çağrıları ve zamanlayıcı mantığı burada.

Fazlar ve LED

  • NOT_INIT → INITED → AUTONOMOUS/TELEOP akışı.
  • LED: Mavi sabit (kimse bağlı değil), Mavi yanıp sönüyor (UI bağlı), Sarı sabit (INIT), Turuncu yanıp sönüyor (Otonom), Yeşil yanıp sönüyor (Teleop).

Kullanıcı geri çağrıları

  • robotInit(): INIT’te bir kez; nesne kurulumları ve kalibrasyon.
  • teleopInit()/teleopLoop(): Joystick ile sürüş; mapping seçimi ve periyodik kontrol.
  • autonomousInit()/autonomousLoop(): Zaman/Adım tabanlı senaryo; sayaçlar.
  • robotEnd(): STOP’ta güvenli temizlik.

Ayrı bir teleopEnd() yoktur; genel temizlik için robotEnd() kullanılır.

Zamanlama ve görevler

ESP32 üzerinde Probot, çok çekirdekli bir görev modeli kullanır: UI/Input ayrı, Kontrol ve Kullanıcı döngüleri ayrı çekirdekte çalışır. Scheduler periyodik güncellemeleri (~20ms) tetikler ve IUpdatable::update(now, dt) sözleşmesiyle çağrı yapar.

  • control::attach(obj): Nesneyi zamanlayıcıya dahil eder; detach ile çıkarılır.
  • Periyot sapması: Süre aşımı durumunda deadlineMiss işaretlenir; performans/log yükünü azaltın.
  • İpucu: Döngülerde engelleyici gecikmelerden kaçının; küçük delay(20) yeterlidir.

Örnek senaryo

// C++
 void teleopInit(){
  // Joystick eşlemesini seç, sayaçları sıfırla
}

void teleopLoop(){
  auto js = probot::io::joystick_api::makeDefault();
  float left  = js.getLeftY();
  float right = js.getRightY();
  chassis.setVelocity(left * 100.0f, right * 100.0f);
  chassis.update(millis(), 20);
  delay(20);
}

void autonomousInit(){
  step = 0; t0 = millis();
}

void autonomousLoop(){
  if(step==0){ chassis.driveDistance(50.0f); step=1; t0=millis(); }
  else if(step==1 && millis()-t0>3000){ chassis.turnDegrees(90.0f); step=2; }
  delay(20);
}

İyi pratikler

  • Teleop/otonom döngülerinde kısa gecikmeler (delay(20)) kullanın; bloklayıcı uzun işlemlerden kaçının.
  • Otonomda basit bir durum makinesi (state machine) kurun; zaman damgası ile adım geçişleri yapın.
  • Setpoint/konfigürasyonları uygun init fonksiyonunda yazın; loop’ta yalnızca güncelleyin.

Scheduler ve Periyotlar

Scheduler, periyodik güncellemeleri tek merkezden tetikler. Varsayılan periyot ~20 ms’tir ve tüm IUpdatable nesnelerin update(nowMs, dtMs) metodu bu ritimde çağrılır.

Kullanım

  • Updatable ekleme/çıkarma: control::attach(obj), control::detach(obj)
  • Global periyot: control::setGlobalPeriodMs(periodMs) ile değiştirilebilir (0 ≠ geçerli).
// robotInit içinde önerilir
#include <probot/core/scheduler.hpp>

void robotInit(){
  // Periyodu 10ms yapmak isterseniz (daha hassas ama daha pahalı)
  control::setGlobalPeriodMs(10); // 0 verilirse yok sayılır

  // Updatable nesneleri ekleyin
  control::attach(&chassis);     // BasicTankDrive, IUpdatable
  // control::attach(&closedLoopMotorLeft);
  // control::attach(&closedLoopMotorRight);
}

void robotEnd(){
  control::detach(&chassis);
}

dt ve performans

  • dtMs, gerçek periyodu temsil eder; PID ve filtrelerde sabit 0.02 yerine dtMs * 0.001f kullanın.
  • Update’lerde uzun işlemler ve yoğun Serial.println periyot sapmasına yol açar; sapma aşırıysa deadlineMiss işaretlenir.

Öneriler

  • Teleop/otonom üst döngülerinde sadece setpoint verin; asıl hesaplar update() içinde çalışır.
  • Periyot küçültüldüğünde (örn. 10ms), update’lerinizi daha da hafifletin; IO’yu kuyruklayın/seyreltin.
  • Gereksiz attach etmeyin; ihtiyaç bitince detach edin.

Joystick API

Probot, oyun kumandası girişini WPILib tarzı bir API ile sunar. Amaç; takımınız hangi kumandayı kullanırsa kullansın, akslar ve butonlar için tek, tutarlı bir arayüz sağlamaktır.

Eksenler ve tuşlar

  • Sol çubuk: getLeftX(), getLeftY()
  • Sağ çubuk: getRightX(), getRightY()
  • Tetikler: getLT(), getRT() (0.0–1.0)
  • Butonlar: getA(), getB(), getX(), getY(), getLB(), getRB(), getBack(), getStart()
  • D-Pad: getDpad() → derece (−1: yok, 0: yukarı, 90: sağ, 180: aşağı, 270: sol)

Mapping seçimi

Farklı gamepad’ler farklı eksen/tuş dizilimlerine sahip olabilir. Mapping, bu farkları tek API altında toplar. Varsayılan eşleme çoğu kontrolcüde çalışır; gerekirse makeXbox(), makeDS4() gibi yardımcılar kullanılabilir. Eşlemeyi teleopInit()’te seçin.

Deadzone ve filtre

  • Merkezde küçük titreşimleri yok etmek için deadzone uygulanır (örn. 0.05).
  • İsteğe bağlı olarak kare veya cubic eğriyle ince ayar yapılabilir.
#include <probot/io/joystick_api.hpp>

probot::io::Joystick js;

void teleopInit(){
  js = probot::io::joystick_api::makeDefault();
  js.setDeadzone(0.05f);
  // Örnek: xbox için
  // js = probot::io::joystick_api::makeXbox();
}

void teleopLoop(){
  float throttle = js.getLeftY();     // -1.0 .. +1.0
  float turn     = js.getRightX();    // -1.0 .. +1.0

  // Tank/arkad sürüşe çevir
  float left  = throttle + turn;
  float right = throttle - turn;

  chassis.setVelocity(left * 100.0f, right * 100.0f);
  delay(20);
}

İpuçları

  • Butonları edge-detect (basıldı/bırakıldı) için son durum takibi yapın.
  • D-Pad’i basit state machine’lerde menü/komut seçimi için kullanın.

Motor/Encoder Soyutlamaları

Çeşitli sürücüler (PWM, CAN, I2C…) ve farklı enkoder tiplerini tek bir ortak arayüz altında topluyoruz. Amaç; aynı kodu farklı donanımlarda çalıştırabilmek ve kapalı çevrim (PID) kontrolü kolayca kullanmak.

Kavramlar

  • IMotor: Düşük seviyede motoru sürer (yüzde/voltaj). Yön invert edilebilir, fren/boşta modu ayarlanabilir.
  • IEncoder: Konum/hız geri bildirimi verir. ticks ve ticksPerRev üzerinden normalize edilir; sıfırlanabilir (zero()).
  • ClosedLoopMotor: IMotor + IEncoder’ı birleştirir; hız ve konum modlarında PID ile sürer. IUpdatable’dır.
  • MotorHandle: Sahiplik (claim/release) katmanı; aynı motorun eşzamanlı iki yerden sürülmesini engeller.
  • MotorGroup: Birden çok motoru tek kanal gibi kullanma (ör. sol taraf 2 motor).

Bağlama ve kalibrasyon

#include <probot/devices/motor.hpp>
#include <probot/devices/encoder.hpp>
#include <probot/controllers/closed_loop_motor.hpp>
#include <probot/core/scheduler.hpp>

// Donanımı oluşturun (örnek: PWM motor + quadrature enkoder)
auto leftMotor = probot::devices::makePwmMotor(5);          // PWM pin
auto leftEnc   = probot::devices::makeQuadratureEncoder(17, 18); // A,B pinleri

ClosedLoopMotor left(leftEnc, leftMotor);
left.setInverted(false);
left.setEncoderPulsesPerRevolution(2048);
left.setGearRatio(10.71f);                        // kutu oranı
left.setWheelCircumferenceMeters(0.319f);         // teker çevresi (opsiyonel)

// PID ayarları (slot 0)
left.pid(0).setGains(0.35f, 0.0f, 0.01f, 0.0f);   // P, I, D, F
left.selectPidSlot(0);

control::attach(&left); // Scheduler periyodik update() çağırır

Kontrol modları

  • Hız (velocity): Sabit bir hız hedefi takip edilir. Sürüş ve besleme mekanizmaları için ideal.
  • Konum (position): Belirli bir rotasyon/mesafe hedefine gider. Kol/eklem, asansör vb. için ideal.
// Hız modu (örnek)
left.setVelocityRpm(300.0f);

// Konum modu (örnek)
left.setPositionTargetRotations(5.0f);   // 5 tur
// Teker çevresi tanımlıysa mesafe hedefi de verilebilir:
// left.setPositionTargetMeters(1.0f);

İyi pratikler

  • Kalibrasyon: ticksPerRev, dişli oranı ve yön (invert) doğru olmalı; aksi halde PID düzgün çalışmaz.
  • Slotlar: Farklı mekanizmalar için farklı PID slotları kullanın; çalışma sırasında slot değiştirilebilir.
  • Güvenlik: Akım/sürüş limiti ve yazılımsal limitler (soft limit) tanımlayın; STOP’ta detach edin.
  • Gruplar: Aynı tarafta birden fazla motor varsa MotorGroup ile tek komut verin; takipçi motorları invert etmeyi unutmayın.
  • Encoder yoksa: IMotor ile açık çevrim sürüş yapılabilir; kapalı çevrim kadar hassas olmayacaktır.
